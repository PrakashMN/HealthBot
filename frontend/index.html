System Architecture

Frontend (Nginx + React)

Static frontend served by Nginx (container). UI uses React via CDN, Chart.js for analytics, and Font Awesome for icons.

Backend (FastAPI + Uvicorn)

Python 3.11 backend built with FastAPI, served by Uvicorn ‚Äî exposes `/chat`, `/symptom-check`, and other API endpoints.

LLM Integrations

AI layer integrates with Groq (preferred) and OpenAI as fallback to generate health-aware responses; keys managed via `.env`.

Primary Data Store (PostgreSQL)

Postgres holds user records, vaccination history, and analytics data; configured as a service in Docker Compose.

Cache & Sessions (Redis)

Redis used for caching and session data to speed up responses and reduce load on the primary database.

Containerization (Docker Compose)

Services (frontend, backend, db, redis) are orchestrated with Docker Compose for local development and testing.

Technical Implementation

Programming Languages

  • Python 3.11 (Backend, AI integration)
  • JavaScript / ES6 (Frontend, client logic)
  • HTML & CSS (Static UI)
  • SQL (PostgreSQL queries)

Frameworks & Libraries

  • FastAPI (Python backend)
  • Uvicorn (ASGI server)
  • React (via CDN for UI components)
  • Chart.js (Charts / analytics)
  • requests, python-dotenv (utility libs)

Databases & Storage

  • PostgreSQL (Primary relational DB)
  • Redis (Caching and sessions)
  • Optional: AWS S3 for file/assets if used in production

APIs & Integrations

  • Groq API (preferred LLM)
  • OpenAI (fallback LLM)
  • Pluggable: Twilio / WhatsApp Business API for messaging
  • Optional: Google Translate or other translation APIs for multilingual support

Cloud & DevOps

  • Docker (container images)
  • Docker Compose (local multi-service orchestration)
  • Nginx (serves static frontend in container)
  • CI: GitHub Actions (recommended)

Monitoring & Analytics

  • Sentry (error tracking)
  • Prometheus + Grafana (optional metrics & dashboards)
  • ELK or cloud logging (centralized logs)

Security & Compliance

Data Protection

  • Environment-based secrets (`.env`) and server-side secret management
  • Encryption in transit (TLS) and encryption at rest where applicable
  • Data minimization and anonymization for analytics
  • GDPR/HIPAA considerations when handling personal health data

Infrastructure Security

  • TLS/HTTPS for all external traffic
  • API key validation and rate limiting for external integrations
  • Use of container isolation and least-privilege service accounts
  • Monitoring and alerting (Sentry / Prometheus) for anomalous activity
